PAGÍNAS WEB: PROTOCOLO

El protocolo HTTPS (HyperText Transfer Protocol Secure) es la versión segura de HTTP, el protocolo principal que se utiliza para intercambiar información entre un navegador web y un sitio web. HTTPS añade una capa de seguridad mediante el cifrado de los datos, lo que resulta fundamental cuando los usuarios envían información sensible, como contraseñas, datos bancarios o información personal al iniciar sesión en distintos servicios en línea.

 

Actualmente, cualquier sitio web, y especialmente aquellos que requieren credenciales de acceso, debería utilizar HTTPS. Los navegadores modernos, como Google Chrome, identifican claramente los sitios seguros mediante un icono de candado en la barra de direcciones y advierten a los usuarios cuando una página no utiliza HTTPS, marcándola como “no segura”.

 

La seguridad de HTTPS se basa en el protocolo TLS (Transport Layer Security), anteriormente conocido como SSL (Secure Sockets Layer). Este protocolo utiliza un sistema de cifrado asimétrico conocido como infraestructura de clave pública. En este sistema intervienen dos claves: una clave pública, que está disponible para los usuarios y se usa para cifrar la información enviada al servidor, y una clave privada, que solo posee el propietario del sitio web y que se utiliza para descifrar esos datos.

 

Gracias a este mecanismo, HTTPS impide que la información transmitida pueda ser leída fácilmente por terceros. En el caso de HTTP, los datos viajan en texto plano y pueden ser interceptados con facilidad mediante herramientas comunes, lo que hace que la comunicación sea muy vulnerable, especialmente en redes Wi-Fi públicas. Por ello, HTTPS es esencial para garantizar la confidencialidad, integridad y seguridad de las comunicaciones en Internet.
